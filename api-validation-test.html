API Validation Documentation

This tool provides a simple interface to test various aspects of the API validation functionality:

Key Features

  • Schema Validation - Verify that input data adheres to expected schemas before reaching API endpoints
  • Security Validation - Check inputs for potential security issues such as SQL injection or XSS vulnerabilities
  • Validation System Status - View current validation rules, mappings, and security configuration

Example: Schema Validation

// Example valid user object
{
  "name": "John Doe",
  "email": "john@example.com",
  "age": 30,
  "role": "user"
}

Example: Security Validation

// Safe input
const userId = 123;

// Suspicious input (potential SQL injection)
const userId = "1; DROP TABLE users;--";
